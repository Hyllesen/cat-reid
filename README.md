# Cat Re-ID

A four-phase pipeline that learns to recognise individual cats from a home camera and identifies them by name in real time.

```
RTSP stream → detect → crop → embed → cluster → identify
```

Built for **Apple Silicon** (Mac Mini M4 / MacBook). Runs YOLO via CoreML (Neural Engine) and MobileNetV3 on MPS for 30+ FPS live inference.

---

## How it works

| Phase | Script | What it does |
|---|---|---|
| 1 – Ingest | `ingest.py` | Watches the RTSP stream, detects cats with YOLO, saves cropped images to `data/raw_crops/` |
| 2 – Extract | `extract_features.py` | Runs all crops through MobileNetV3-Large to produce L2-normalised 1280-dim embeddings → `embeddings.pkl` |
| 3 – Cluster | `cluster.py` | Groups embeddings with DBSCAN, organises images into `data/organized/cluster_N/`, writes `cat_map.json` and `centroids.pkl` |
| 4 – Identify | `identify.py` | Runs the live stream again: detects, embeds, compares against cluster centroids, overlays the cat's name (or "New Visitor") |

Phases 1–3 are a one-time setup. Phase 4 is the live system.

---

## Requirements

- macOS with Apple Silicon (MPS + CoreML)
- Python 3.9+
- An IP camera that exposes an RTSP stream

## Setup

```bash
# 1. Create and activate a virtual environment
python3 -m venv .venv
source .venv/bin/activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Configure
cp .env.example .env
# Edit .env and set RTSP_URL to your camera's stream address
```

## Running the pipeline

### Phase 1 — Collect training crops

```bash
python ingest.py
```

Point your camera at the cats and let it run. Every detection with ≥ 70% confidence is saved as a JPEG crop in `data/raw_crops/`. Press **`q`** or **Ctrl-C** to stop. Aim to collect at least a few dozen images per cat across different times of day.

### Phase 2 — Extract embeddings

```bash
python extract_features.py
```

Processes all crops in `data/raw_crops/` and saves `embeddings.pkl`. Takes a few seconds even for hundreds of images on MPS.

### Phase 3 — Cluster and learn identities

```bash
python cluster.py
```

Groups similar embeddings with DBSCAN. Any cluster with more than 5 images is treated as a "resident cat". Each cluster gets a default name (`Unknown Cat 0`, `Unknown Cat 1`, …) in `cat_map.json`.

**Name your cats** by editing `cat_map.json` directly:

```json
{
  "2": {
    "name": "Mochi",
    ...
  }
}
```

> ⚠️ This step *moves* images from `data/raw_crops/` into `data/organized/`. It cannot be undone without re-running Phase 1.

### Phase 4 — Live identification

```bash
python identify.py
```

Opens a preview window showing each detected cat's name in real-time. A green box means a matched resident; orange means "New Visitor". Press **`q`** or **Ctrl-C** to stop.

## Configuration

All options are set via `.env`. Every variable has a sensible default except `RTSP_URL`.

```bash
# Required
RTSP_URL=rtsp://192.168.1.100:554/stream

# Detection
CONFIDENCE_THRESHOLD=0.7   # Minimum YOLO confidence to save/identify a crop
FRAME_SKIP=1               # Process every Nth frame (increase to reduce CPU load)

# Stream reliability
MAX_RECONNECT_ATTEMPTS=10
RECONNECT_DELAY=5          # Seconds between reconnect attempts

# Clustering (Phase 3)
DBSCAN_EPS=0.20            # Max cosine distance between neighbours
DBSCAN_MIN_SAMPLES=5       # Min neighbours to form a core point
RESIDENT_MIN_IMAGES=5      # Min cluster size to be a "resident cat"

# Identification (Phase 4)
MATCH_THRESHOLD=0.25       # Cosine distance below which a cat is "matched"

# Debug logging
DEBUG=False
```

**Tuning tips:**
- If Phase 3 creates too many small clusters, raise `DBSCAN_EPS` (try `0.25`).
- If unrelated cats are merged into one cluster, lower `DBSCAN_EPS` (try `0.15`).
- If Phase 4 misidentifies cats, lower `MATCH_THRESHOLD` to be more strict.
- If known cats are labelled "New Visitor", raise `MATCH_THRESHOLD` slightly.

---

## Re-training

When new cats appear regularly, re-run Phases 1–3 to rebuild the knowledge base:

```bash
# 1. Collect new crops (Phase 1)
python ingest.py

# 2. Re-extract embeddings from ALL crops
python extract_features.py

# 3. Re-cluster (re-creates cat_map.json and centroids.pkl)
python cluster.py

# 4. Re-name any new clusters in cat_map.json, then restart Phase 4
python identify.py
```

```
├── extract_features.py     Phase 2 — embedding extraction

├── centroids.pkl           Generated by Phase 3
└── data/
    ├── raw_crops/          Phase 1 output → consumed by Phase 3

        ├── cluster_1/
        └── cluster_noise/  Unclustered images
```
